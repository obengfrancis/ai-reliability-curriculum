# User Profile Service

A small Java HTTP client that retrieves and manages user profile data
from a remote API, with a local on-disk cache for repeat reads. This
codebase is the working artifact for **Lab 1: "AI Wrote It Wrong"**.

## What the service does

`UserProfileService` exposes four operations:

- **`fetchUserProfile(userId)`** — GETs `/users/{id}` from the remote
  service and returns the parsed JSON body.
- **`updateProfile(userId, updates)`** — POSTs a JSON body to
  `/users/{id}` and returns whether the server accepted it.
- **`getCachedProfile(userId)`** — reads a previously-cached profile
  from local disk; returns `null` when no cache entry exists.
- **`loadProfileBatch(userIds)`** — fetches multiple profiles in a
  single call, returning a map of the ones that loaded successfully.

The `Main` class boots a small in-process HTTP server that imitates the
remote API, then exercises each operation against it. This lets you see
the service work end-to-end before you start the fault analysis.

## How the code was produced

This codebase was generated by an AI coding assistant in response to the
prompt:

> *"Write a Java service that fetches user profile data from a REST API
> at a configurable base URL, supports profile updates via POST, caches
> profiles to local disk, and can load multiple profiles in a batch."*
 
 The service compiles, runs, and produces correct output on the happy path. 
 Your job in Lab 1 is to identify where it will fail in the field, and why.

## Build and run

Requirements: JDK 17 or later, Maven 3.6+.

```bash
mvn compile           # build the classes
mvn exec:java         # run the demo against the in-process API
```

The demo runs in a few seconds. Expected output: each of the four
operations exercised, with a printed result for each.
